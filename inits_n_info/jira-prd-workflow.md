# Jira PRD Workflow

## PRD Review Process

1. **Create PRD** in `docs/APP_PRD.md`
2. **Create Jira Task** for PRD Review
3. **Move to "In Review"** when ready for approval
4. **Get approval** from stakeholder
5. **Move to "Done"** when approved

## Workflow States

| State | Meaning |
|-------|---------|
| To Do | Not started |
| In Review | Awaiting approval |
| In Development | Being built |
| Testing | Being tested |
| Done | Completed |

## PRD Checklist

Before moving to "In Review":
- [ ] All sections of PRD filled
- [ ] User stories defined
- [ ] Technical stack confirmed
- [ ] Data model documented
- [ ] UI/UX requirements clear
